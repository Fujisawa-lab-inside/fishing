# Stage 20 cross-condition holdout plan

更新日: 2026-07-16

S02ブラウザ比較のA案採用により、表示する全1時間時刻のsnapshotを保持し、欠落時刻の直線補間を禁止する方針を固定した。次に必要なのは、異なる条件の間を補間した流速地図が直接solverと一致するかの検証である。

## 推奨する最初の試験

河口堰開度を最初に検証する。

- 閉鎖0%と全開100%の2本を補間材料にする。
- 50%開度を50:50で推定する。
- 既に計算済みのreference S02（50%）を、補間には使わない正解データとして比較する。
- 比較時刻は-12、-11、-10、-9、-8時間。
- 河口全域、河口堰、曲川・遠賀川合流部、魚道の4範囲を確認する。
- 最悪時刻について直接計算、補間結果、速度ベクトル誤差の地図を作る。

河口堰を先にする理由は、50%が両端の正確な中点で判定しやすく、堰と魚道周辺は線形補間が破綻しやすい重要領域だからである。ここで不合格なら、残りの潮位・遠賀川・西川・曲川の8本を計算する前に、線形応答基底方式を見直せる。

## 合格候補基準

5時刻と4範囲のすべてで次を満たすことを候補基準とする。

- 速度ベクトルRMSE: `0.01 m/s`以下
- 流速MAE: `0.005 m/s`以下
- 流向誤差p95: `5°`以下（流速`0.02 m/s`以上のcell）

範囲maskは実行前にSHA-256で固定する。一部時刻または一部範囲だけの合格は公開結果に使わない。

## 計算範囲候補

閉鎖・全開の各条件について、model時刻-24〜-8の16物理時間を4時間ずつ4区間に分ける。合計8 jobで、-12〜-8時間の毎時snapshotを保存する。

reference S01+S02実績からの単純換算は合計約15.6 runner時間、2系列を並列実行した理想経過時間は約7.8時間である。極端な堰条件では計算速度が変わるため保証ではない。各jobは5時間で停止し、自動retry、失敗後の継続、別条件restartの流用を禁止する。

## 次の判断

- A（推奨）: 河口堰だけを先に検証する8-jobの停止状態contractを作り、実行承認は別に判断する。
- B: 5入力すべてを一括検証する40-jobの停止状態contractを作り、実行承認は別に判断する。

A案が採用され、河口堰だけの8-job contract、fail-closed runner、activation専用workflow、実行判断画像を停止状態で準備した。authorization、gate、activationは存在せず、数値計算は0 stepである。次の判断は、この8 jobを承認後24時間以内に一回限り実行するかである。物理計算、S03、公開接続、`main`反映はまだ承認されていない。
