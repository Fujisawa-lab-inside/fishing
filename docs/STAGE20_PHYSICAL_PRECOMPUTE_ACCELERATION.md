# Stage 20 物理事前計算の高速化

更新日: 2026-07-15

承認されたメッシュv2の採用後、物理計算を実行せずに事前計算経路を高速化した。

- 50,199セルのメッシュv2に物理runnerを移植した。
- 数値流束、静水圧再構成、境界条件、摩擦、CFL規則を変えず、mesh indexと逆面積をキャッシュし、face-to-cell集約を`numpy.bincount`へ置換した。
- 300ステップの決定的な非物理合成試験では、旧メッシュ＋旧kernel比で19.85倍、同じメッシュv2上のkernel単体で1.22倍だった。
- mesh v2上で旧kernelと新kernelの最大相対状態差は`6.41e-16`で、許容`1e-12`以内だった。
- 11本の36時間応答基底jobは、出力digest付きcheckpointから再開でき、完了jobを再計算しない制御経路に更新した。
- メッシュv2の4地点地図と証拠manifestを作る一回限りworkflowを準備した。未承認時は数値依存の導入前に停止する。

旧物理checkpointへの合成試験倍率の適用では、1条件×物理10分は約26.4分と推定される。ただし実測ではなく、最大60分で未完了停止する可能性がある。

次の判断は、GitHub提供Linux x86 runnerでこの1条件×10分を承認後24時間以内に一回だけ実行するかである。自動再試行はなく、authorizationとgateは選択Aの明示承認後にだけ作成する。現時点では物理計算、workflow、公開シミュレータ接続、`main`反映のいずれも実行していない。
