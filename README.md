## v4.8.0 統一水面仕様

- `onga_unified_water_domain_v480.js` を最終水面・流体ドメインとして読み込みます。
- `onga_unified_boundary_v480.js` を水面・陸地境界および岸上釣り座候補の最終レイヤーとして読み込みます。
- ユーザー指定画像の青塗り領域だけを唯一の正解水面として扱います。
- 橋1・橋2・橋3と2つの合流部地形先端、計5点の実測座標で水面マスクを地理参照します。
- 水面判定、ヒートマップ生成・クリップ、流体計算waterセルは同じ `contains(lat,lng)` と同じ正規化waterセル配列を参照します。
- 診断格子では、水面判定セル・ヒートマップ領域セル・流体waterセルの差分を `0` セルとします。
- 左上の水域は橋下で主水域へ接続し、全体を1つの連結水域として扱います。
- 黄色の橋は水面上の構造物であり、橋下の水面・流れ・ヒートマップを分断しません。
- 赤線の河口堰は水理構造物であり、陸地ギャップを作りません。
- 開境界は河口・潮汐側1か所、西川・遠賀川本流・曲川の上流側3か所です。
- 4つの内部参照断面は位置と端点だけを保持し、固定流向・固定符号・長い河道軸を保持しません。
- 上げ潮・下げ潮による流向は、時刻ごとの水位・流量条件からポテンシャル流ソルバーが決定します。
- 岸上釣り座候補は統一水面の水面・陸地境界だけから抽出し、画像端の開境界は除外します。
- 魚道中心は `33°53′20.8″N 130°40′28.5″E`、局所流方向は北東45°です。
- 魚道固有の釣り座除外は設定せず、魚道影響は統一水面内だけで有効にします。

## 検証

- JavaScript構文確認: `node --check`
- 統一ドメイン単体テスト: waterセル数、開境界4、無方向参照断面4、魚道座標、安全除外0 mを検証
- 上げ潮・下げ潮テスト: 同じwaterセル配列を共有しつつ、境界水頭とポテンシャル場が時刻条件に応じて変化することを検証
- 診断格子: `differenceCells = 0`

## v4.7.8 の修正点

- 評価点群から作る水面領域を連結成分へ分割します。
- 主要水域または高信頼 `water_polygon` に接続しない孤立水面候補を除外します。
- 陸地内で孤立して見える誤認識水面を、主要水域外として除外しやすくしました。
- 除外後の主要水域外縁だけを青点線で表示します。
- 釣り座候補も、除外後の主要水域外縁サンプルを優先します。
- `state.photoSampleStatus` に孤立除外セル数を表示します。

## v4.7.7 の修正点

- `onga_pointcloud_water_region_v477.js` を追加しました。
- スコア計算用の水面評価点群から水面領域を生成します。
- 評価点群から生成した水面領域の外縁を、通常表示で青点線として描画します。
- `water_polygon` 外縁は補助扱いとし、主表示は評価点群外縁へ移行します。
- ホットスポットの水面ターゲットは、これまで通り評価点を使います。
- 釣り座候補は、評価点群外縁のサンプルを優先します。
- 橋・河口堰・画面端・開境界に近い外縁サンプルは釣り座候補から除外します。
- legacy_shoreline は通常表示から外し、評価点群外縁とGeoJSON水面判定を優先します。

## 注意

このシミュレータは釣行判断用の簡易モデルです。安全確認の代替ではありません。増水，濁流，流木，足場冠水，強風時は釣行しないでください。
